DMD：One-step Diffusion with Distribution Matching Distillation

arXiv 2311.18828 · CVPR 2024 · Tianwei Yin (MIT + Adobe)

1.1 核心数学：KL 散度的 score 差分梯度

设学生生成器 G（参数 θ）将高斯噪声 z 映射到图像/视频 x̂ = G(z;θ)，其输出分布为 p_G。目标是最小化：

目标
minθ D_KL(p_G ‖ p_data)

对 θ 求梯度，利用 log p 对参数的链式法则展开：

KL 梯度（关键推导）
∇θ D_KL = Ex̂~p_G[ (∇x log p_G(x̂) − ∇x log p_data(x̂)) · ∂x̂/∂θ ]

= Ex̂~p_G[ (s_fake(x̂,t) − s_real(x̂,t)) · ∂G/∂θ ]

s_fake = ∇x log p_G（学生输出分布的 score，用一个独立扩散模型拟合）
s_real = ∇x log p_data（真实数据分布的 score，用教师模型初始化后固定）
🔑
核心技巧：两个 score 的差天然是对 G 的梯度信号，不需要显式的密度函数。s_real 用教师模型初始化（权重复制），然后冻结；s_fake 需要跟随 G 的输出分布实时更新。

1.2 总损失（含回归正则项）

DMD 总损失
L_DM = Ex̂~G(z), t~U[0,T][ ‖s_fake(x̂ + σ_t·ε, t) − s_real(x̂ + σ_t·ε, t)‖² ]

L_reg = Ez, x*=DDIM_teacher(z)[ ‖G(z) − x*‖² ]

L_total = L_DM + λ·L_reg   （λ 论文中约 0.05~0.1）

1.3 Fake Score 模型的训练

s_fake 必须实时跟踪当前 G 的输出分布，否则梯度估计偏差会导致 G 更新方向错误。用标准 score matching 在 G 的输出上训练：

Fake Score 更新目标
L_score = Et, x̂=G(z), ε~N(0,I)[ ‖s_fake(x̂ + σ_t·ε, t) + ε/σ_t‖² ]

等价于：让 s_fake 学会从加了噪声的学生输出预测原始噪声（标准去噪目标）

1.4 完整训练算法（伪代码）

DMD 训练循环 
# 初始化
G          = copy_weights(teacher)   # 学生生成器，LoRA 或全参数
s_fake     = copy_weights(teacher)   # fake score 网络，全参数
s_real     = copy_weights(teacher)   # 真实 score 网络，冻结
precomputed = ddim_sample(teacher, dataset)  # 提前生成 (z, x*) 配对

for step in range(max_steps):

    # ── 阶段1：更新 G ──────────────────
    z = sample_noise(batch_size)
    x_hat = G(z)                         # 一步生成

    t = sample_timestep()
    ε = sample_noise(batch_size)
    x_noisy = x_hat + σ[t] * ε

    grad_dm = s_fake(x_noisy, t) - s_real(x_noisy, t)
    # 用 grad_dm 当作梯度（不反传 score 网络）
    x_hat.backward(grad_dm)

    x_star = lookup(precomputed, z)    # 取出对应的教师多步输出
    loss_reg = mse(x_hat, x_star)
    loss_reg.backward()

    optimizer_G.step()

    # ── 阶段2：更新 fake score（更频繁）──
    for _ in range(k_critic):          # k_critic ≥ 1
        z = sample_noise(batch_size)
        x_hat = G(z).detach()         # 不回传到 G
        t = sample_timestep()
        ε = sample_noise(batch_size)
        x_noisy = x_hat + σ[t] * ε

        loss_score = mse(s_fake(x_noisy, t), -ε / σ[t])
        loss_score.backward()
        optimizer_fake.step()

DMD2：三大改进的精确机制

arXiv 2405.14867 · NeurIPS 2024 Oral · Tianwei Yin et al.

2.1 改进① — 去除回归损失，引入 GAN 判别器

DMD 的 L_reg 需要提前用教师运行数万次 DDIM 50步采样，在大分辨率下成本不可接受。DMD2 用 GAN 判别器替代：

DMD2 总损失（无 L_reg）
L_total = L_DM + λ·L_GAN

L_DM = E[ ‖s_fake(G(z)+noise, t) − s_real(G(z)+noise, t)‖² ]

L_GAN(G) = −Ex̂~G[ D(x̂) ]
L_GAN(D) = Ex~data[max(0, 1−D(x))] + Ex̂~G[max(0, 1+D(x̂))]

判别器 D 可复用 s_fake 的特征提取部分，不引入新参数
🔍
为什么 GAN loss 效果更好？GAN 判别器直接在真实数据上训练，对 s_real 估计偏差不敏感（它不依赖 s_real）。而 L_reg 的梯度方向依赖教师 DDIM 轨迹，这条轨迹并不能代表真实数据分布的全貌。

2.2 改进② — 两时间尺度更新规则（Two Time-Scale Update Rule）

去掉 L_reg 后，系统变得不稳定：G 一旦跑偏，s_fake 来不及追踪，产生错误梯度，形成正反馈发散。解法是让 s_fake 的更新速度远快于 G：

学生生成器 G

η_G = 5e-5
更新步：每个 step 1次
参数量：LoRA (rank 64)

更新慢，保证 s_fake 有余量追踪当前 p_G

Fake Score / Critic

η_fake = 4e-6 ~ 1e-5
更新步：每个 step 多次
参数量：全参数

更新快（约 G 的 10~100 倍），实时追踪 p_G 变化

⚠️
工程陷阱：两个网络的学习率比值比绝对值更重要。比值建议从 10 开始，观察 fake score loss 是否先于 G loss 收敛——如果 fake score loss 还未收敛时 G 已经开始更新，说明比值太小。

2.3 改进③ — Backward Simulation（反向模拟）

DMD 训练时学生输入 = 纯高斯噪声 z，但多步推理的第 2、3、4 步输入是前一步学生输出再加噪。这个分布差异（train-test mismatch）使多步蒸馏质量降级。

Backward simulation 在训练时模拟完整的多步推理链：

Backward Simulation 伪代码（来自 NVIDIA Cosmos 实现） 
def training_step_critic(batch, student, critic, cfg):
    # 反向模拟：从当前时间步推若干步
    x = batch['latent']
    with torch.no_grad():
        for t in get_backward_steps(cfg.num_backward_steps):
            # 学生去一步噪声
            x_denoised = student(x, t, condition)
            # 重新加噪到下一个时间步（backward simulation 核心）
            x = re_noise(x_denoised, t_next)

    # 在 backward-simulated 分布上训练 critic
    loss_critic = score_matching_loss(critic, x, t)
    return loss_critic

def training_step_generator(batch, student, critic, teacher, cfg):
    # student 生成（此处可以是多步）
    z = sample_noise(batch)
    x_hat = multi_step_student(student, z, cfg.num_student_steps)

    # teacher CFG 目标（distill CFG 进 student）
    t = sample_timestep()
    x_noisy = add_noise(x_hat, t)
    s_real_cond   = teacher(x_noisy, t, condition=cond)
    s_real_uncond = teacher(x_noisy, t, condition=uncond)
    s_real = s_real_uncond + cfg_scale * (s_real_cond - s_real_uncond)

    # 分布匹配梯度
    s_fake = critic(x_noisy, t)
    grad_dm = s_fake - s_real

    # GAN 梯度
    loss_gan = -discriminator(x_hat).mean()

    # 合并（DM 梯度直接注入，GAN 梯度正常反传）
    x_hat.backward(grad_dm * cfg.dm_weight)
    loss_gan.backward()

    return grad_dm.norm(), loss_gan

2.4 CFG 蒸馏（额外 2× 加速）

DMD2 将教师的 Classifier-Free Guidance（CFG）行为蒸馏进学生，使学生推理时不需要跑两次前向（条件 + 无条件）：

CFG 蒸馏目标
s_real_cfg(x,t,c) = s_teacher(x,t,∅) + ω·(s_teacher(x,t,c) − s_teacher(x,t,∅))

用这个 CFG-combined score 作为 s_real 目标，学生学会"一次前向输出 CFG 效果"
推理时不再需要无条件前向，额外节省 2× 计算

2.5 TrigFlow 参数化（来自 Cosmos 实现）

Cosmos DMD2 实现使用 TrigFlow 作为统一参数化（来源于 sCM 论文），对视频 DiT 的时间步嵌入和噪声 schedule 均有影响：

TrigFlow 时间步参数化
σ(t) = sin(t)  （代替 DDPM 的指数噪声 schedule）
α(t) = cos(t)  t ∈ [0, π/2]

student 推理时间步：手动定义，如 [T, 0.5T, 0.1T] 三步
不依赖第三方 scheduler（避免 mismatch）

DMD/DMD2 适配自有模型：完整改动清单

3.1 模型组件结构

适配 DMD2 需要维护三个网络对象，均从同一套 base model 权重初始化：

模型初始化结构 
# 你的 base model 类（继承关系）
class DMD2VideoModel(DistillationMixin, TrigFlowMixin, YourBaseVideoDiT):
    """
    YourBaseVideoDiT  : 原始视频 DiT 架构（不改 forward 逻辑）
    TrigFlowMixin     : 提供时间步采样、噪声 schedule 函数
    DistillationMixin : 提供 training_step_generator / training_step_critic
    继承顺序很重要（MRO），DistillationMixin 放最前
    """
    def __init__(self, cfg):
        super().__init__(cfg)

        # Teacher：完整权重，全程冻结
        self.teacher = load_pretrained(cfg.ckpt).eval()
        for p in self.teacher.parameters(): p.requires_grad = False

        # Student（= self）：LoRA 或全参数
        self._setup_lora(rank=64, alpha=8)   # 可选

        # Fake Score / Critic：完整权重，全参数训练
        self.critic = load_pretrained(cfg.ckpt)
        # Critic 不用 LoRA，需要充足表达能力追踪 p_G

        # （可选）单独的 GAN 判别器头，或复用 critic 的特征
        self.discriminator = DiscriminatorHead(
            in_dim=cfg.hidden_dim, # 复用 critic 倒数第 N 层特征
        )

3.2 需要在 base model 上实现的新方法

sample_student_timestep()
手动定义学生推理的时间步序列，替代 scheduler。对于 1 步生成，返回固定 t=T（最大噪声级别）；4 步生成返回 [T, 0.7T, 0.4T, 0.1T]。
backward_simulation(student, x_T, n_steps)
从纯噪声出发，运行 n_steps 步学生去噪，每步输出重新加噪到下一时间步。返回各时间步的 (x_t, t) 对，用于训练 critic。这是解决 train-test mismatch 的关键。
compute_cfg_score(x_noisy, t, cond)
调用 teacher 两次（有条件 + 无条件），合并为 CFG score：s = s_uncond + ω·(s_cond − s_uncond)。这个 CFG score 是 L_DM 的蒸馏目标，学生学成后推理不再需要 CFG。
training_step_generator() / training_step_critic()
两阶段交替训练的入口函数。Generator 步：critic 冻结，计算 L_DM + L_GAN 更新 student。Critic 步：student 冻结，在 backward-simulated 样本上更新 critic。

3.3 噪声 Schedule 适配

⚠️
常见坑：如果你的 base model 使用 DDPM/EDM/DDIM 的噪声参数化，而直接套 DMD2 的 TrigFlow 时间步，会导致 score 估计错误。需要确认：
  • s_fake 和 s_real 用相同的噪声参数化（同样的 σ_t 定义）
  • 学生和教师的 timestep embedding 语义一致（t=999 对两者意味着同样的噪声级别）
  • 如果 base model 是 flow matching（Wan2.1, HunyuanVideo），TrigFlow 兼容性更好

3.4 适配检查清单

  • ✓从 base model 复制三份权重：student（LoRA）/ teacher（冻结）/ critic（全参）
  • ✓实现 backward_simulation，确认多步展开的 re-noise 使用与 teacher 相同的 σ_t
  • ✓Teacher 调用两次（cond / uncond）合成 CFG score 作为 s_real
  • ✓Critic 学习率设为 student LR 的 10~100 倍
  • ✓GAN 判别器复用 critic 特征（避免引入新参数）
  • △如果 base model 是视频 DiT（非图像），backward_simulation 中的 re-noise 需按时序维度操作
  • △视频长序列（>30K tokens）建议先做 SLA2 再做 DMD2，否则 teacher 调用成本极高
  • ✗不要用第三方 scheduler（如 DPM-Solver）控制 student 的推理步骤——手动定义时间步序列
  • ✗不要在 critic 上使用 LoRA——它需要全参数表达能力追踪 p_G

SLA：Sparse-Linear Attention 精确机制

arXiv 2509.24006 · 清华 + 伯克利 · GitHub: thu-ml/SLA

4.1 核心发现：注意力权重的二元结构

SLA 的出发点是对 Wan2.1 视频 DiT 注意力矩阵做奇异值分解（SVD）后的实证发现：

高秩稀疏部分（约5%的块）

幅值大的注意力权重块，对应"当前 token 高度关注的目标区域"。这部分的秩较高，低秩近似误差大，必须精确计算（FlashAttention）。

低秩稠密部分（约95%的块）

接近均匀分布的小权重块，秩极低（可用 1~2 个奇异向量近似）。用线性注意力（低秩特征映射）近似，误差可控。

4.2 块划分与三分类算法

将 QK^T 按 block 划分（块大小 BLKQ = BLKK = 64 tokens）：

SLA 块分类算法 
def classify_blocks(Q, K, kh_pct=0.05, kl_pct=0.10, blk=64):
    """
    kh_pct: 关键块比例 (top-k high)  —— 论文 kh%=5%
    kl_pct: 次要块比例 (top-k low)   —— 论文 kl%=10%
    剩余 85% 的块为可忽略块
    """
    B, H, N, D = Q.shape
    Q_blk = Q.reshape(B, H, N//blk, blk, D)  # 分 block
    K_blk = K.reshape(B, H, N//blk, blk, D)

    # 估计每个 QK 块的幅值（用块内 max 或 mean 做代理）
    # 不需要完整计算 A = softmax(QK^T)，用近似统计即可
    block_scores = estimate_block_importance(Q_blk, K_blk)
    # shape: (B, H, N/blk, N/blk)

    n_blocks = (N // blk) ** 2
    n_critical  = int(n_blocks * kh_pct)   # 5%
    n_marginal  = int(n_blocks * kl_pct)   # 10%

    # 按 block_scores 从大到小排序
    sorted_idx = argsort(block_scores, descending=True)
    critical_mask  = sorted_idx[:n_critical]   # 精确计算
    marginal_mask  = sorted_idx[n_critical:n_critical+n_marginal]  # 线性近似
    # 其余 85% 直接跳过（置 0）

    return critical_mask, marginal_mask

4.3 线性注意力核（Marginal 块的近似）

SLA 用 ELU 特征映射将 softmax 注意力分解为线性乘积，避免 O(N²) 计算：

线性注意力公式
φ(x) = ELU(x) + 1   （保证非负，近似 softmax 的指数增长特性）

Attn_linear(Q,K,V) = φ(Q) · [φ(K)ᵀ · V]

计算顺序：先算 φ(K)ᵀ·V（O(N·D²)，算一次全局共享）
再对每个 Q_i 查询：φ(Q_i)·[φ(K)ᵀ·V]（O(N·D)）

总复杂度：O(N·D²) 远小于 O(N²·D)（当 N >> D 时）
💡
精度说明：ELU 核与真实 softmax 的近似误差在低权值区域可控。对于 marginal 块（幅值本就小），近似误差对最终输出的影响远小于直接跳过（skip）带来的信息丢失。这正是 SLA 超越纯稀疏方法的原因。

4.4 Fine-Tuning：恢复质量的蒸馏微调

替换注意力后直接推理会有质量下降，用少量数据做注意力输出蒸馏：

SLA 微调损失
L_SLA = Evideo~D, layer l[ ‖SLA_output(Q,K,V) − FullAttn_output(Q,K,V)‖² ]

· 对每一层的注意力模块分别施加损失
· FullAttn 输出作为软标签（教师），SLA 模块作为学生
· 只优化 SLA 相关参数（块阈值，线性核权重）
· 不更新 FFN / LayerNorm / 其他层

SLA2：Learnable Routing + QAT 精确机制

arXiv 2602.12675 · 清华 + 伯克利

5.1 改进① — 可学习路由器（替代启发式阈值）

SLA 的块分类依赖"先算出注意力权重，再按幅值决定路由"——这是一个循环悖论：你必须先做计算才能决定如何计算。SLA2 用一个轻量 MLP 路由器在前向前预测路由：

SLA2 可学习路由器 
class BlockRouter(nn.Module):
    def __init__(self, dim, blk=64):
        super().__init__()
        # 极轻量：对 Q 和 K 的块均值做 MLP
        self.router = nn.Sequential(
            nn.Linear(dim * 2, dim // 4),  # Q_mean concat K_mean
            nn.ReLU(),
            nn.Linear(dim // 4, 1),           # 输出每对块的路由 logit
        )

    def forward(self, Q, K, blk=64):
        # 压缩表示：块内平均池化
        Q_blk = Q.reshape(..., N//blk, blk, D).mean(dim=-2)  # (B,H,N/blk,D)
        K_blk = K.reshape(..., N//blk, blk, D).mean(dim=-2)

        # 所有 QK 对的路由分数
        Q_exp = Q_blk[:, :, :, None, :].expand(..., N//blk, D)
        K_exp = K_blk[:, :, None, :, :].expand(N//blk, ..., D)
        feat = torch.cat([Q_exp, K_exp], dim=-1)  # (B,H,N/blk,N/blk,2D)

        logits = self.router(feat).squeeze(-1)    # (B,H,N/blk,N/blk)

        if self.training:
            # 软路由（可微分）
            return torch.sigmoid(logits)
        else:
            # 硬路由（Straight-Through）
            hard = (logits > 0).float()
            return hard + logits.sigmoid() - logits.sigmoid().detach()
            # STE：前向用 hard，反向用 sigmoid 梯度

5.2 改进② — 忠实的稀疏-线性分解

SLA 中"marginal 块用线性注意力"并非 softmax 的严格分解。SLA2 提出一个数学上更忠实的公式：

SLA2 忠实分解
A = softmax(QKᵀ/√D) = α·A_sparse + (1−α)·A_linear

Output = α·SparseAttn(Q,K,V) + (1−α)·LinearAttn(Q,K,V)

α ∈ ℝH 是逐注意力头的可学习标量（H = num_heads）
随训练自动调整稀疏分支和线性分支的权重
💡
为什么更忠实？SLA 的"3 类块"是一个离散的划分，不能微分优化。SLA2 的 α 让模型端到端决定"对每个注意力头，稀疏注意力应该贡献多少"，更贴近对 softmax 的连续近似。

5.3 改进③ — QAT（量化感知训练）

注意力模块的计算瓶颈之一是内存带宽（Q/K/V 矩阵读写），低比特量化可进一步降低带宽需求。SLA2 用 QAT 避免直接量化的精度损失：

SLA2 QAT 实现（Fake Quantization） 
def quantize_tensor(x, scale, bits=8):
    # 模拟 INT8 量化（forward only）
    qmin, qmax = -128, 127
    x_q = torch.clamp(torch.round(x / scale), qmin, qmax)
    x_dq = x_q * scale   # 反量化回 FP16
    # STE：前向用量化值，反向当作恒等函数
    return x + (x_dq - x).detach()

def sla2_attention(Q, K, V, router, alpha, training=True):
    if training:
        # Fake Quantization：训练时模拟量化误差
        scale_Q = Q.abs().max() / 127
        scale_K = K.abs().max() / 127
        Q_q = quantize_tensor(Q, scale_Q)   # 量化 Q
        K_q = quantize_tensor(K, scale_K)   # 量化 K
        # V 保持 FP16（精度更敏感）
    else:
        # 推理时用真实 INT8 kernel（SageAttention2++）
        Q_q, K_q = sage_int8_quantize(Q, K)

    # 路由决策
    route = router(Q_q, K_q)   # (B,H,N/blk,N/blk)，软/硬路由

    # 稀疏注意力（精确计算，仅 route=1 的块）
    attn_sparse = sparse_flash_attention(Q_q, K_q, V, mask=route)

    # 线性注意力（近似，仅 route=0 的块）
    attn_linear = linear_attention_elu(Q_q, K_q, V, mask=1-route)

    # 忠实分解合并（alpha 逐头可学习）
    return alpha * attn_sparse + (1 - alpha) * attn_linear

6.1 代码级改动：替换 Attention Module

SLA 的改动局限在每个 DiT block 的 attention 子模块，不触碰 FFN / 归一化 / 位置编码：

替换策略：模块级热插拔 
from sparse_linear_attention import SparseLinearAttention  # thu-ml/SLA

def replace_attention_with_sla(model, config):
    for name, module in model.named_modules():
        if isinstance(module, YourAttentionClass):
            # 构建 SLA 模块，继承原始 Q/K/V/O projection 权重
            sla_module = SparseLinearAttention(
                head_dim   = config.head_dim,   # 128
                topk       = config.topk,       # 0.05（kh%=5%）
                feature_map= 'elu',            # ELU+1 核
                BLKQ       = 64,
                BLKK       = 64,
            )
            # 复制 Q/K/V/O 投影权重（关键：不随机初始化）
            sla_module.copy_qkvo_weights(module)
            # 替换
            set_module_by_name(model, name, sla_module)
    return model

6.2 微调流程

准备微调数据集：SLA 原论文用 20,000 条 5秒 480p 视频（自采集）。实际上对质量无严格要求——目的是提供多样的时空 token 分布让 attention 看到。可用公开数据集（Panda-70M、WebVid 子集）或内部视频数据替代。
冻结非 SLA 参数，运行双前向：每个 batch 跑两次前向：① 完整注意力（Full Attention，教师）；② SLA 注意力（学生）。收集每层的注意力输出差，用 L2 损失优化 SLA 参数。只需 2000 步。
逐层验证质量：在微调中间（如 500/1000/2000 步）生成样本，计算 FVD 或 CLIP 相似度与原模型对比，确认质量无损。通常 kh%=5% 时 2000 步后质量完全恢复。
（SLA2）加入路由器训练：路由器 MLP 参数从零初始化，随微调一起训练。α 初始化为 0.5（均等混合），学习后通常收敛到每头不同的值，反映不同注意力头的稀疏偏好。
（SLA2）QAT 阶段：在常规 SLA2 微调收敛后，加入 fake quantization，再训练 500~1000 步让模型适应量化误差。推理时替换为真实 INT8 kernel（SageAttention2++）。

6.4 适配检查清单

  • ✓确认 base model 的 attention 是标准 QKV 格式（非 grouped query / sliding window 等变体）
  • ✓SLA 替换时复制原始 Q/K/V/O projection 权重，不随机初始化
  • ✓微调时冻结所有非 SLA 参数（FFN / LN / embedding 等）
  • ✓每 500 步生成样本验证 FVD，防止过拟合导致质量下降
  • △如果序列长度 < 5K，SLA 加速收益有限（O(N²) 计算量本就不大）——考虑提高分辨率或帧率后再用
  • △SLA2 路由器初始化用全零（输出 sigmoid=0.5，对应均等混合），避免早期路由偏向极端
  • △QAT 阶段建议先在小规模数据上验证量化误差可接受后，再全量训练
  • ✗不要对 V 矩阵做量化（精度最敏感，量化收益远小于风险）

问题原因解法
