研究全景与脉络

视频扩散模型的两大加速瓶颈

视频生成面临两个独立但互补的效率瓶颈：采样步数过多（DMD/DMD2 解决），以及单步注意力计算量过大（SLA/SLA2 解决）。两条线并行发展，最终在"超少步 + 超低注意力开销"的联合优化上汇聚。

瓶颈一：采样步数

标准扩散模型需要 20-50 步去噪，每步都要完整跑一遍 DiT 前向，视频生成延迟无法接受。

→ DMD / DMD2 将步数压缩到 1-4 步

瓶颈二：注意力复杂度

视频 token 序列长度高达 30K+，自注意力 O(N²) 占总计算量 60% 以上，即便 1 步推理也慢。

→ SLA / SLA2 将注意力降低到 O(N) 量级

联合收益

两种加速正交，可叠加：DMD2 步数 ×N 倍加速，SLA2 每步注意力 ×18.6 倍加速，端到端加速乘积显著。

→ 研究前沿：联合蒸馏

DMD：One-step Diffusion with Distribution Matching Distillation

arXiv 2311.18828 · CVPR 2024 · Tianwei Yin et al. (MIT + Adobe Research)

核心动机：为什么不能直接蒸馏？

直觉上可以用 score distillation sampling (SDS) —— 但 SDS 的梯度需要对真实分布计算 score，而真实分布未知，只有教师模型近似。直接让学生输出逼近教师某个确定性轨迹（regression loss）会导致 mode averaging（图像模糊）。DMD 的核心洞见是：不要在样本空间对齐，而是在分布空间对齐。

数学基础：近似 KL 散度

设学生生成器 G 输出分布为 p_G，教师真实数据分布为 p_data。目标是最小化：

目标函数
minG D_KL(p_G ‖ p_data)

KL 散度对 G 参数的梯度为：

KL 梯度展开
∇θ D_KL(p_G ‖ p_data)
= Ex~p_G [ ∇x log p_G(x) − ∇x log p_data(x) ] · ∂G/∂θ

= Ex~p_G [ s_fake(x,t) − s_real(x,t) ] · ∂G/∂θ
💡
关键洞见：KL 梯度等于两个 score function 之差。只要能分别估计 p_G 和 p_data 的 score，就不需要显式密度。 两个 score 各用一个独立的扩散模型来参数化：s_fake 在学生生成的样本上训练，s_real 在真实数据上训练（直接用教师权重初始化）。

完整训练流程

z ~ N(0,I)
🎓
学生生成
x̂ = G(z; θ)

📊

两个损失的作用

分布匹配损失 L_DM

L_DM = E[‖s_fake(G(z),t) − s_real(G(z),t)‖²]

驱动学生输出的整体分布向真实数据分布靠拢。使学生能覆盖真实数据的多个模态，不会模糊。

回归损失 L_reg

L_reg = E[‖G(z) − DDIMteacher(z)‖²]

防止 L_DM 训练初期不稳定导致的梯度爆炸，作为正则化项。需预先运行教师多步采样生成"伪标签"数据集。

Fake Score 模型的训练

这是 DMD 的细节难点。s_fake 需要实时跟踪学生 G 生成的分布变化，因此要与 G 交替训练：

s_fake 更新目标（score matching）
L_score = Et,x̂~p_G,ε [ ‖s_fake(x̂ + σ_t ε, t) − ε‖² ]
其中 x̂ = G(z) 是当前学生的输出，σ_t 是扩散噪声尺度

DMD 的局限

⚠️
昂贵的预生成数据集：回归损失需要教师用 DDIM 多步采样生成一个大规模噪声→图像配对数据集，对大分辨率（如 SDXL 1024²）成本极高。

Fake score 滞后问题：s_fake 无法完美实时追踪 p_G 的变化，会引入误差梯度，导致学生早期训练不稳定。

DMD2：Improved Distribution Matching Distillation

arXiv 2405.14867 · NeurIPS 2024 Oral 🔥 · Tianwei Yin et al.

DMD2 在 DMD 的基础上提出三个独立但协同的改进，彻底解决了 DMD 的三个痛点。

改进 ①

去除回归损失和预生成数据集，改用 GAN 判别器作替代正则化

改进 ②

两时间尺度更新规则，解决 fake score 训练滞后导致的不稳定

改进 ③

反向模拟（Backward Simulation），解决训练-推理分布不匹配，并支持多步生成

改进① — GAN 损失替代回归损失

DMD2 彻底去除了 L_reg，无需预计算教师轨迹数据集。取而代之引入一个判别器 D，在真实图像和学生生成图像之间做区分：

GAN 损失（Hinge 形式）
L_GAN(G) = −Ex̂~p_G[D(x̂)]
L_GAN(D) = Ex~p_data[max(0,1−D(x))] + Ex̂~p_G[max(0,1+D(x̂))]
🔍
为什么 GAN loss 能替代回归 loss？
回归损失的本质是"样本级对齐"，GAN 损失是"分布级对齐"。GAN 判别器的梯度信号更接近 DMD 的分布匹配哲学，且无需预生成数据，直接在真实数据上训练判别器即可。

总训练损失变为：

DMD2 总损失
L_total = L_DM + λ · L_GAN
λ 为 GAN 损失权重超参数，论文中约 0.1

改进② — 两时间尺度更新规则 (Two Time-Scale Update Rule)

DMD 的 fake score 不稳定根源在于：学生 G 更新太快，fake score 来不及跟上，导致 KL 梯度估计偏差。DMD2 采用不对称学习率：

学生 G 的学习率

η_G = 1e-5（慢）

G 参数更新步幅小，保证 fake score 有时间追踪当前分布。

Fake Score 的学习率

η_fake = 1e-4（快 ×10）

fake score 更新远快于 G，确保在 G 每次更新时都能准确估计当前 p_G 的 score。

改进③ — 反向模拟 (Backward Simulation)

DMD 训练时用噪声 z 作为学生输入，但推理时多步采样的中间输入（来自前一步学生输出加噪）与之不同，形成 train-test mismatch。DMD2 通过 backward simulation 解决：

🎲
纯噪声 z
第 T 步
🎓
学生第1步
x̂₁ = G(z,t₁)
📡
重新加噪
x̃₁ = 加噪(x̂₁,t₂)
🎓
学生第2步
x̂₂ = G(x̃₁,t₂)
📊
计算损失
每步均计算
L_DM + L_GAN

这样在训练中真实模拟了多步推理的输入分布，student 在每一步接受的输入与推理完全一致。同时这一机制自然支持 1 步、2 步、4 步等灵活步数设置。

SLA：Beyond Sparsity in Diffusion Transformers via Fine-Tunable Sparse-Linear Attention

arXiv 2509.24006 · Jintao Zhang et al. (清华 + 伯克利)

问题背景：视频 DiT 的注意力困境

Wan2.1 等视频 DiT 在处理 5 秒 480p 视频时，时空 token 序列长度高达 30,000+。标准全注意力（Full Attention）复杂度 O(N²)，注意力模块占单步推理时间的 60% 以上，成为核心瓶颈。

此前纯稀疏注意力方法（如 SpargeAttn）在稀疏比例超过 70% 后质量急剧下降，原因在于：被跳过的"边际权重"块虽小但非零，积累误差显著。

核心观察：注意力权重的二元结构

🔬
关键发现：视频 DiT 的注意力权重矩阵 A 可以分解为两个部分：

① 稀疏高秩部分：少量（约 5-30%）幅值大的权重块，包含主要语义信息，秩高，必须精确计算。
② 稠密低秩部分：大量接近均匀分布的小权重块，整体呈极低秩结构，可用线性注意力（低秩分解）近似。

三分类注意力块

SLA 将注意力矩阵按 block（例如 64×64 token block）划分，依据块内注意力权重幅值将每个 QK 块分为三类：

🔵 Critical（关键块）

权重幅值 ≥ 阈值 τ_high
用 精确 FlashAttention 计算
复杂度 O(N_critical²)

🟢 Marginal（边际块）

权重幅值在 [τ_low, τ_high)
用线性注意力核近似（ELU 或 ReLU 特征映射）
复杂度 O(N·d)

⚫ Negligible（可忽略块）

权重幅值 < τ_low
直接跳过，贡献为 0
复杂度 O(1)

线性注意力近似

对 Marginal 块，利用核技巧将 softmax 注意力分解为特征映射乘积：

线性注意力近似（ELU 特征映射）
Attn_linear(Q,K,V) = φ(Q) · (φ(K)ᵀ V)
先算 φ(K)ᵀV（O(N·d·d) 一次），再对每个 Q 查询（O(N·d)）
φ(x) = ELU(x) + 1 确保非负性

Fine-Tuning 恢复质量

替换注意力后直接推理质量下降，SLA 用少量视频数据做微调恢复：

蒸馏微调损失
L_finetune = E[‖ SLA_output − FullAttn_output ‖²]
用原始完整注意力输出作为软标签，对 SLA 模型做特征级蒸馏

实验中仅用 20,000 个 5 秒 480p 视频，微调数千步即可完全恢复生成质量。

实验结果（Wan2.1-1.3B）

95%
注意力计算量减少
13.7×
注意力模块加速
2.2×
端到端视频生成加速
≈0
FVD 质量损失

工程实现：SageSLA

基于 SageAttention 实现的定制 CUDA 核，统一处理三类 block 的分支路径，支持 bfloat16 和 block-level 动态调度。端到端加速（2.2×）远小于注意力单模块加速（13.7×）的原因在于：video DiT 中 FFN 和归一化层也占相当比例，注意力并非唯一瓶颈。

SLA2：Sparse-Linear Attention with Learnable Routing and QAT

arXiv 2602.12675 · Jintao Zhang, Haoxu Wang, Kai Jiang et al. (清华 + 伯克利)

SLA 的三个残留缺陷

缺陷一：启发式路由

SLA 依赖注意力权重幅值来划分三类块，但这是一个事后估计——要先算出权重才能决定如何算，存在循环依赖，且阈值手工设定不具普适性。

缺陷二：分解不忠实

SLA 的"边际块用线性注意力"并非数学上对原始 softmax 注意力的严格近似，两者在某些 token 模式下存在系统性偏差。

缺陷三：无量化

SLA 仍使用 BF16 精度。注意力的 Q/K/V 占用大量带宽，低比特量化可进一步降低内存访问瓶颈，但直接量化会引入较大误差。

改进① — 可学习路由器 (Learnable Router)

SLA2 引入一个轻量级路由网络 R，在正向传播前预测每个 QK 块应走稀疏还是线性分支，无需实际计算注意力权重：

路由决策（软路由，训练时）
r_{ij} = σ( R(Q_i, K_j) ) ∈ [0, 1]
Attn_{ij} = r_{ij} · SparseAttn(Q_i,K_j) + (1−r_{ij}) · LinearAttn(Q_i,K_j)
路由决策（硬路由，推理时）
r_{ij} = 1 if R(Q_i,K_j) ≥ 0.5 → 精确计算
r_{ij} = 0 if R(Q_i,K_j) < 0.5 → 线性近似
💡
路由器的设计：R 是一个极轻量的 MLP（几个线性层 + 激活），输入为 Q_i 和 K_j 的压缩表示（通过池化降维），参数量相对整个 DiT 可忽略不计。训练用 Straight-Through Estimator 保证梯度流通过硬路由决策。

改进② — 忠实的稀疏-线性分解

SLA2 提出了一个数学上更严格的分解方式。核心思路是：softmax 注意力权重可以分解为一个"高注意力聚焦部分"与"低注意力背景部分"，对应的 value 聚合可以分别计算并通过可学习混合比 α 合并：

忠实分解（Faithful Decomposition）
A = softmax(QKᵀ/√d) = α · A_sparse + (1−α) · A_linear
Output = α · (A_sparse · V) + (1−α) · LinearAttn(Q,K,V)
α 是逐层/逐头可学习标量，随训练自动调整稀疏与线性的权重

改进③ — 量化感知训练 QAT

SLA2 在微调时同步加入 QAT，将 Q/K/V 矩阵量化到 INT8，用 fake quantization 节点模拟量化误差，让模型学会在低精度下保持精度：

综合视角：联合方向与研究机会

两条线的正交性

DMD/DMD2 和 SLA/SLA2 从不同维度压缩了视频生成的计算量，且几乎不互相干扰：

联合训练的挑战

损失冲突

DMD2 的分布匹配损失梯度作用在输出 token 上，SLA2 的蒸馏损失作用在注意力中间层。联合优化时梯度尺度需要仔细权衡。

教师选择

联合蒸馏的"教师"是谁？可选：① 完整步数 + 完整注意力教师；② 少步但完整注意力教师。后者将 SLA2 的问题简化为注意力替换。

训练顺序

先做 SLA2（注意力压缩），再在压缩后的模型上做 DMD2（步数压缩）可能比联合训练更稳定，且每阶段目标清晰。

反向模拟兼容性

DMD2 的 backward simulation 在 SLA2 的近似注意力下是否稳定？需要验证多步展开时近似误差不会累积爆炸。
